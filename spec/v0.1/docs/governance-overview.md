# PLS Governance Overview

**Date:** 2026  
**Author:** AI MacRae  

---

## 1. Purpose

The Provenance Ledger Standard (PLS) provides a **governance framework** for high-impact AI systems. It defines the minimum machine-readable record that such systems must maintain as a condition of deployment.

The PLS is not a standalone standard. It is the **evidence layer** within a broader AI assurance architecture.

---

## 2. Core Principles

The PLS is built on five core principles:

| Principle | Description |
| :--- | :--- |
| **Evidence** | Trust is not a governance strategy. Evidence is. |
| **Transparency** | Systems must show their work, not just declare their intent. |
| **Accountability** | Provenance enables attribution and corrective action. |
| **Auditability** | Independent verification must be possible. |
| **Proportionality** | Requirements scale with system impact (PLS-Lite → PLS-Critical). |
